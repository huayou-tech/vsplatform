## 一、整体定位（一句话卖点）

**这是一套经过十年生产环境验证的「企业级视频监控平台」完整源码**，覆盖从底层 SDK、设备接入、流媒体转发、质量诊断到客户端/网页端展示的全链路，并以 SIP/GB28181、ONVIF、RTSP 等标准协议打通多厂商设备。可面向三类买家：
1. 需要**二次开发**安防监控系统的集成商；
2. 学习**大型 C++ 系统架构**的高校/培训机构；
3. 想快速搭建**视频质量诊断 / 流媒体服务**的创业团队。

## 三、各模块详解 & 卖点提炼

### 1. VSNetSDK —— 设备接入 SDK ⭐⭐⭐⭐⭐
- **位置**：`VSNetSDK/`（含 `doc/` 完整中文接口文档 HTML）
- **功能**：`VS_NET_Login` / `VS_NET_RealPlay` / `VS_NET_StopRealPlay` / `VS_NET_Logout` 等标准接口，设备登录、实时预览、云台控制、回放。
- **卖点**：**开箱即用的设备对接层**，文档齐全（自带 39 个 HTML 接口说明），集成商最需要的"第一层"。
- **依赖**：`Common/network4app/`

### 2. VSPlaySDK —— 播放解码 SDK ⭐⭐⭐⭐
- **位置**：`VSPlaySDK/`（185 个 `.h` 头文件 + 59 个 `.htm` 文档）
- **功能**：音视频解码、渲染、播放控制。
- **卖点**：与 VSNetSDK 配套，形成"接入+播放"完整闭环。

### 3. VSMNetSDK / HYVideoCapSDK —— 媒体与采集 SDK ⭐⭐⭐⭐
- **位置**：`VSMNetSDK/`、`HYVideoCapSDK/`（477 个 dll + 381 个 h，含 33 个 `.idl` 接口定义）
- **卖点**：提供 IDL 接口定义，适合做 COM/ActiveX 形态交付。

### 4. VQDService —— 视频质量诊断服务 ⭐⭐⭐⭐⭐（高溢价）
- **已有文档**：`README_VQDService.md`
- **功能**：编码质量（H.264/H.265）、传输质量（抖动/丢包/带宽）、播放质量（清晰度/色彩/卡顿/音视频同步）三维诊断。
- **卖点**：基于 FFmpeg + OpenCV，<100ms 单路诊断、100+ 路并发，直播质量保障场景刚需。
- **注意**：根目录目前只有 README，需把实际源码（应在某子目录）归拢进这个新仓库。

### 5. VSPCMService —— 设备管控核心服务 ⭐⭐⭐⭐⭐（高溢价）
- **已有文档**：`README_VSPCMService.md`
- **技术**：IOCP 高并发、自定义线程池、内存池、SOAP/WebService、SQL Server/MySQL。
- **卖点**："十年生产验证"的企业级后台骨架，可当毕业设计/架构范本。
- **目录结构**已在 README 描述（MemPool / threadpool / tools / webservice）。需把源码归拢。

### 6. HYSipServer —— SIP / GB28181 国标服务 ⭐⭐⭐⭐⭐（高溢价）
- **位置**：`HYSipServer/`
- **子模块**：`VSPSipService`（SIP 服务）、`VSPSipSDK`、`VSPSipJoinSDK`、`VSPSRTPSDK`（RTP/PS 封装）、`VSPSipMonitor`（监控端）、`SysConfig`（DES/Base64 加密）
- **第三方库**：`jrtplib-3.9.1`、`jthread-1.3.1`、`libosip2_3.6`（在 Common）
- **卖点**：**国标 28181 接入**是国内安防项目入门最大门槛，此模块直接打通。

### 7. HYRTService / RTSPClientSDK —— 流媒体服务 ⭐⭐⭐⭐
- **位置**：`HYRTService/`、`RTSPClientSDK/`（含 live555 风格 `*.hh` 头）
- **卖点**：RTSP 转发与客户端拉流，可做流媒体网关。

### 8. HYDisplay 多媒体客户端 ⭐⭐⭐⭐
- **位置**：`HYDisplay_多媒体客户端/`
- **子模块**：`HYClient`（主客户端）、`HYMClient`、`HYOcxClient`（OCX 网页控件）
- **卖点**：含网页 OCX 控件，能直接嵌入浏览器做演示，降低买家试用门槛。

### 9. HYRegisterLib / HYRegisterLib4Decoder —— 授权注册库 ⭐⭐⭐
- **卖点**：MD5 + 机器码 + 注册码机制，本身就是可单独售卖的"软件加密/授权"通用组件。

### 10. Common 公共库 ⭐⭐⭐（基础依赖）
- **包含**：`CommonFunc`（通用函数）、`conhash`（一致性哈希）、`tinyxml`、`CxImage`（图像）、`jpeglib`、`DuiLib_Ultimate`（界面库）、`boost_1_60_0`、`live555`、`ffmpeg`、`libosip2`、`network4app`、`Register`（注册 UI）
- **策略**：作为各 SDK 仓库的 `third_party/` 子模块或独立依赖仓库，避免重复打包。

### 11. HYCLog / LEARNING_MATERIALS / Python 自动化
- `HYCLog`：轻量日志组件，可免费开源引流。
- `LEARNING_MATERIALS`：架构/概念/习题/实战文档，作为**教程包溢价**（配合视频课卖更高价）。
- Python 脚本（`csdn_auto_publisher.py`、`open_crawl_auto_system.py`、`multi_platform_publisher.py` 等）：已用于自动发文引流，可作为**免费引流仓库**吸引目标买家关注付费 C++ 模块。

## 四、分拆上传策略（行动清单）

### 原则
1. **一个仓库 = 一个可独立编译/使用的产品**，降低买家决策成本。
2. **通用依赖（Common/HYCLog）单独成仓**，用 Git submodule 引用，避免每个仓都塞几百 MB。
3. **高溢价模块（VQD/VSPCM/SIP/SDK）单独成仓并写好 README + 示例**，这是收入主力。
4. **学习资料 + Python 脚本免费开源**，用作 GitHub 引流，README 里互相链接导流到付费仓。

## 五、上线前必做清单（重要）

1. **移除非卖代码**：删除 `*.obj / *.tlog / *.dll（第三方）/ __pycache__ / .suo / .sdf` 等编译产物，用 `.gitignore` 过滤。
2. **补齐 VQD/VSPCM 源码**：根目录只有 README，需把对应源码子目录归拢进新仓（先确认源码实际位置）。
3. **统一 LICENSE 策略**：付费仓建议用**自定义「源代码授权协议」**（禁止转售/需署名），不要用 MIT（否则别人可白嫖）。免费仓才用 MIT/Apache。
4. **第三方库合规**：`jrtplib/jthread/libosip2/live555/ffmpeg/boost/DuiLib` 多为 LGPL/BSD，注意保留其原始 LICENSE，避免整包闭源违规。
5. **脱敏**：检查 `csdn_config.json`、`SysConfig` 里是否含真实账号/IP/密码，上传前清理。
6. **每个仓配 `README + 编译步骤 + 一个最小 demo`**，付费转化率高 3 倍以上。
